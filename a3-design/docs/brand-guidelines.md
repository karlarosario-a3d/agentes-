# Brand Guidelines

Identity guidance for A3Data, derived from the DataViz Guide (Figma node `2:21`). Institutional copy is paraphrased, not reproduced.

## Brand

- **Name:** A3Data (A3DATA Consultoria S/A).
- **Domain:** data consultancy; the analyzed manual is the **DataViz Guide** for dashboards and data visualization.

## Visual principles (paraphrased)

- **Clarity over decoration.** Visualizations should simplify complex information and make patterns, trends and relationships easy to read.
- **Clean, professional look.** A modern sans-serif type system (Segoe UI) keeps interfaces legible.
- **Consistency.** Standardized color, typography and spacing create a uniform experience across dashboards.
- **Engagement through good design.** Well-designed visuals hold attention and communicate more memorably — without adding noise.

## Logo

- Two primary lockups: **main mark on light background** and **main mark on dark background**, plus approved **variations** (`needs-review` for the complete variation inventory and clear-space/min-size specs).
- **Rules (paraphrased):** use only the original brand files; never alter, add to, redraw, or recreate the mark's drawing or typography.

## Iconography

- **Library:** Phosphor Icons 2.1.
- **Style:** Stroke / Regular weight (per component metadata).
- Keep icon style consistent; do not mix icon families.

## Usage do's and don'ts

**Do**
- Use the documented palette and the base-8 spacing system.
- Pair sentiment color with text/icon for accessibility.
- Use original logo files only.

**Don't**
- Introduce colors, fonts, or spacing outside this system without documenting why.
- Alter or recreate the brand mark.
- Rely on color alone to convey state.
